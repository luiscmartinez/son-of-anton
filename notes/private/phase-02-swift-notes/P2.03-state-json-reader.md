# P2.03 — StateJsonReader (TS-developer field notes)

What a TS reviewer needs to know to read `StateJsonReader.swift` and
`ActivityState.swift` honestly. This is not a Swift tutorial; only the
concepts that appear in this PR.

## `enum` with raw values — different from TS enums

```swift
enum ActivityState: String, Equatable, Codable {
    case idle = "idle"
    case implementing = "implementing"
    case runningTests = "running-tests"
    case celebrating = "celebrating"
}
```

- `: String` is a **raw-value enum**: each case has an associated literal
  string. `ActivityState(rawValue: "running-tests")` returns `.runningTests`
  or `nil`. TS analogue: a `const` union with a string-to-symbol map.
- Conformance to `Codable` is automatic for `String`-raw enums: the encoded
  form is the raw value. We override `init(from:)` below to add the fallback.
- `Equatable` is automatic for enums without associated values; we still
  list it explicitly because Swift requires every protocol to be named.

## Custom `init(from:)` — the unknown-state → idle trick

```swift
init(from decoder: Decoder) throws {
    let raw = try decoder.singleValueContainer().decode(String.self)
    self = ActivityState(rawValue: raw) ?? .idle
}
```

`Decoder` is Swift's serialization input port. `singleValueContainer()` is
"give me the bare scalar this position holds." We pull a `String`, look it
up in our raw-value table, and fall back to `.idle` on miss. This is the
"decode any unrecognized string as idle" path the ticket calls for — it's
strictly cheaper than the alternative of throwing and recovering at a
higher layer (rejected in the ticket's Refactor section).

TS analogue would be a Zod schema with `.catch("idle")`. Same shape.

## `Codable` struct + `keyDecodingStrategy = .convertFromSnakeCase`

```swift
private struct StatePayload: Decodable {
    let schemaVersion: Int
    let activityState: ActivityState
    let updatedAt: String
}

let decoder = JSONDecoder()
decoder.keyDecodingStrategy = .convertFromSnakeCase
let payload = try decoder.decode(StatePayload.self, from: data)
```

- `Decodable` here means "I can be constructed from a decoder." `Codable` is
  `Encodable & Decodable`. We only read, so `Decodable` is enough internally.
- `.convertFromSnakeCase` rewrites `schema_version` → `schemaVersion` etc.
  during key lookup. This applies to **keys only** — enum raw values are
  not touched, which is why `"running-tests"` stays kebab-case in the JSON
  and the enum's raw value matches that literal string.
- Extra JSON keys (`hp`, `hp_overlay`, `source_event`) are silently ignored
  by `Decodable` — there is no `strict` mode like Zod's. That is the
  forward-compat behavior P2.02's contract clause asks for.

## `Result<Success, Failure>` vs `throws`

Public API surface:

```swift
static func read(at path: String) -> Result<StateSnapshot, StateReadError>
```

Swift has both error patterns. `throws` is idiomatic for "I might fail and
the caller probably uses `do/catch`." `Result` is idiomatic for "the
caller wants to pattern-match on success vs. specific failure cases
without `do/catch` ceremony." Renderer code (P2.07) needs to branch on the
exact failure case (file-not-found, schema-newer, malformed) to pick the
right tooltip and visual, so `Result` keeps the call site cleaner:

```swift
switch StateJsonReader.read(at: path) {
case .success(let snap): render(snap)
case .failure(.fileNotFound): showIdleBaseline()
case .failure(.schemaNewer(let got, let expected)): showSchemaNewerVisual(...)
case .failure(.malformed), .failure(.schemaMissingOrInvalid): showMalformedVisual()
}
```

`StateReadError.schemaNewer(got: Int, expected: Int)` is an enum case
**with associated values** — closer to a TS discriminated-union variant
that carries payload than to a plain enum member.

## `enum` as a namespace

```swift
enum StateJsonReader {
    static func read(at path: String) -> Result<...> { ... }
}
```

`StateJsonReader` declares no cases; it exists only to host `static`
functions. This is the Swift idiom for "namespace, not value" — like
`namespace Foo { ... }` in TS or a module with only named exports. We use
this when there is no per-instance state to carry.

## File-not-found vs malformed

`Data(contentsOf:)` throws an `NSError`; we discriminate by
`NSCocoaErrorDomain` + `NSFileReadNoSuchFileError`. The `FileManager`
double-check below is a belt-and-suspenders for cases where the OS surfaces
a different errno but the file really is absent. Anything else from this
code path is classified as `malformed` so the renderer never crashes on a
partially-written or corrupt file. The schema-version check runs *before*
full decoding so the precise `schemaMissingOrInvalid` / `schemaNewer`
errors win over the generic `malformed` bucket.

## `EXPECTED_STATE_SCHEMA_VERSION`

Compile-time `let` at file scope — the Swift equivalent of an exported
const. Phase 03 will bump this to `2` when the renderer learns a newer
schema; a single-line constant is the cheapest possible bump surface and
trivial to grep.
