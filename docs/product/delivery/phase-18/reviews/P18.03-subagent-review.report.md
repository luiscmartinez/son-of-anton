**Advisory Observations**

1. **`updatePullRequestBody` does not re-send `base`** (`pr-metadata.ts:1214`). That function is called from `advanceToNextTicket` and only updates the body. If an operator manually changes the GitHub PR base between `open-pr` and `advance`, the body-only refresh will not re-correct it. This is deliberate (the correct recovery path is `restack`), but it means base-role discipline is not enforced at the `advance` step. Out of scope for P18.03 and noted only for completeness.

2. **Test 2 body assertion is narrower than test 1.** Test 2 asserts `stringContaining('- stacked base branch: \`release-next\`')` but does not also assert the URL contains `/blob/main/`. Test 1 covers that URL regression independently. The coverage is complete across both tests, but a reviewer reading test 2 in isolation would not see the URL invariant tested. This is a test-legibility note, not a coverage gap.
