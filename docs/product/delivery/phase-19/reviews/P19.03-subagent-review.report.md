**Advisory Observations**

- **Soft size gate creates routing ambiguity.** The Scope section (line 30–33 of quality-control/SKILL.md) says "Do not hard-gate solely on size when the operator is already asking for quality-control capture." This softens the "small enough for one bounded fix commit" criterion in a way that an operator seeking to avoid standalone PR triage could exploit — asking for QC capture when the fix is medium-sized and routing discipline is the wrong one. If P19.04 is going to tighten routing discipline, this soft gate may be revisited then; flagging it so the P19.04 spec can decide whether to harden it.

- **Human verification timing is under-constrained relative to the commit.** Step 5 (human verification) is required "before recording the review gap" (i.e., before step 7), but there is no explicit constraint that human verification must occur before the commit in step 6. An operator could commit first and then produce human verification. This is within the invariant (both are required before ledger append) but the Workflow's narrative sequence implies verification before commit, and making that explicit would prevent a plausible mis-reading.

- **No test asserts the ban on `bun run deliver quality-control`.** Both skill files explicitly forbid this command form, but neither test in `p19-03.test.ts` checks for the prohibiting language. If the ban language were accidentally removed from both files, tests would still pass. This is low-risk given the single repo, but the ban is a structural invariant that could be cheaply tested with a `not.toContain('bun run deliver quality-control')` assertion.

- **`toContain('qc')` in test 2 (line 35) is a loose two-character match.** The assertion would pass on any prose that happened to contain "qc" as a substring in an unrelated context. In the current SoA skill text the only occurrences are the alias section heading and the trigger line, so there is no false-positive risk today — but the assertion provides weaker discoverability proof than checking for the alias heading or trigger string explicitly (both of which are already checked by line 34 and line 37–38).
