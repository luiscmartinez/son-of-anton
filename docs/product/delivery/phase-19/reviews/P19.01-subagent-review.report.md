**Advisory Observations**

- **Preserve test covers only one file's content.** The preserve test pre-creates `ledger.jsonl` with custom content and verifies it byte-for-byte. It does not place custom content in `README.md` or `promotion-queue.md` before sync and verify those survive unchanged. Because the `[ ! -e ]` guard is identical for all three files, the implementation is correct, but a future change that accidentally breaks the guard for a specific file would go undetected for the two unchecked files. Adding a parallel assertion (write sentinel content into all three, rerun, verify all three unchanged) would close this gap.

- **Fresh-creation assertions are existence-only.** Both tests check `existsSync(...)` but not file content on initial copy. If `cp` somehow produced a zero-byte file, the tests pass. This is unlikely in practice but means template content drift (e.g., someone empties a template file) is not caught by the test suite at sync time.

- **Partial-source failure leaves an empty destination directory.** If `.son-of-anton/docs/template/review-gaps/` exists but is missing one of the three expected files, `copy_review_gap_scaffold` creates `docs/product/review-gaps/` via `mkdir -p`, then exits the entire script when `cp` fails on the missing source file. Subsequent reruns reproduce the same failure. The empty destination directory is left behind. This is fail-loud behavior (acceptable) but the residual empty directory could confuse operators. The ticket's rationale does not require handling this case, so it belongs here rather than as a blocking finding.

- **`ledger.jsonl` template has a trailing blank line.** The file contains one blank line (not strictly empty). This is valid as an empty JSONL file and does not affect any downstream tooling described in the ticket.
