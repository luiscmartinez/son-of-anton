### Diff context

The diff expands `SubagentRunnerOutcome` to include `'deferred'`, adds `SUBAGENT_LEDGER_SCHEMA_VERSION = 1`, and adds four optional fields to `SubagentRunnerInvocation`: `schemaVersion?: number`, `primaryAgent?: string`, `runnerSelfReport?: string | null`, `fallbackFrom?: SubagentRunnerKind | null`. `validateRunnerArtifact` validates these fields when present (integer check, string check, string-or-null check, runner-kind-membership-or-null check) but does not back-fill them on output — the output shape exactly matches the input shape (only the documented optional fields appear when input has them). Three reader helpers (`getPrimaryAgent`, `getRunnerSelfReport`, `getFallbackFrom`) materialize defaults at read time. `buildRunnerInvocation` accepts the four new fields via `BuildRunnerInvocationOptions` and conditionally spreads them onto the returned invocation. The test file `tools/delivery/test/subagent-runner.test.ts` covers: all four outcome values round-trip; all four new fields round-trip; legacy row parses with helper-defaults; unknown future `schemaVersion: 999` parses without throwing.
