### Actionable findings

**Finding 1 — `revealPetFolder` does not ensure the directory exists before opening**
File: `apps/menubar/Sources/MenubarMenu.swift`, `revealPetFolder(_:)` (~line 113)

What's wrong: `openLogFolder` calls `fileManager.createDirectory(at:withIntermediateDirectories:)` before `workspace.open(...)` precisely because the folder may not exist on first launch. `revealPetFolder` skips this step. On a machine where Codex has never been run, `~/.codex/pets/` will not exist; the `NSWorkspace.open` call silently fails with no user feedback.

Which clause: Ticket outcome — "A new menu item, **Reveal pet folder**, opens `~/.codex/pets/` in Finder." A silent no-op is not "opens in Finder."

Concrete fix:
```swift
@objc func revealPetFolder(_ sender: Any?) {
    try? fileManager.createDirectory(at: petFolderURL, withIntermediateDirectories: true)
    workspace.open(petFolderURL)
}
```
`fileManager` is already injected on the type; same pattern as `openLogFolder`. The fix is two lines.
