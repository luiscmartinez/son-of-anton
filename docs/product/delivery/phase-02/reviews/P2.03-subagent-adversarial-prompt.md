### Invariants to hold

1. For any input where `JSONSerialization` parses the file as a JSON object AND `schema_version` is an `Int <= EXPECTED_STATE_SCHEMA_VERSION` AND `activity_state`/`updated_at` are strings, `StateJsonReader.read(at:)` returns `.success`; the returned `snapshot.activityState` equals one of the four floor enum cases iff the raw string is exactly `"idle"`, `"implementing"`, `"running-tests"`, or `"celebrating"`, and `.idle` for every other string value.
2. `StateJsonReader.read(at:)` returns `.failure(.schemaNewer(got: N, expected: 1))` iff `schema_version` is an `Int` with `N > EXPECTED_STATE_SCHEMA_VERSION`; it returns `.failure(.schemaMissingOrInvalid)` iff `schema_version` is absent or not decodable as `Int`; it returns `.failure(.fileNotFound)` iff the path does not exist; it returns `.failure(.malformed)` iff the file exists, was read successfully, but is not parseable as a JSON object.
3. The orderings in invariant 2 are mutually exclusive: a missing file never returns `.malformed`; a schema-newer payload never returns `.malformed` even when other fields are wrong; a schema-missing payload never returns `.malformed`.
