**Finding 1 — Desaturate silent color fallback violates Invariant 3**
File: `apps/menubar/Sources/MenubarRenderer.swift`, `desaturate(_:)` (lines 122–138)
What is wrong: When `cgImage(forProposedRect:...)`, `filter.outputImage`, or `ciContext.createCGImage` returns nil, `desaturate()` returns the original colored `NSImage`. In `.desaturated` mode the caller (`renderedCurrentFrame()`) passes this colored image directly to `sink()`, silently emitting a frame that contradicts the renderer's own `currentVisualModeForTesting` and Invariant 3.
Which clause: Invariant 3 broken; also clause 3 ("spec-permits-real-bug") — the fallback is implicitly permitted by the ticket's "or equivalently" wording around desaturation, but it defeats the entire purpose of the "early failure visual" mode.
Fix recommendation: On desaturation failure, either (a) log `NSLog("MenubarRenderer: desaturate failed — emitting plain gray fallback")` and return a synthesized gray `NSImage` of the same size (constructed via `NSImage(size:)` + `NSBezierPath.fill()` in gray), or (b) return `nil` from `desaturate` and have `renderedCurrentFrame()` return nil on filter failure, skipping the sink call rather than emitting a colored frame. Option (b) produces a frozen last-known-good-gray frame; option (a) always emits something gray. Either is better than silently colored.
